Vue项目实践-打造智能数据商场后台管理系统

发表于 2018-02-08

Smart Data Market 以下简称（SDMK）是 TalkingData 作为数据行业的先行者推出的智能数据服务商城。SDMK 在之前有一个1.0版的后台管理系统。但是随着业务和需求的增加，以及公司前端技术栈的统一。我们使用 Vue 重构了之前的系统，命名为 SDMK-ADMIN，同时也根据需求增加新功能。下图是 SDMK-ADMIN 的基本页面样式

目录结构

SDMK-ADMIN 使用 vue-cli 进行项目初始化，然后根据实际需求进行了一些调整，上图是 SDMK-ADMIN 的目录结构。首先介绍一下src目录重里面的内容。

views和components

从上面的网站侧边栏导航可以看到，我们的后台在设计页面的时候，是按照不同的功能管理模块来组织页面的。因此，views 目录中的每一条子目录对应着页面上的一个管理模块。components 是存放页面组件的目录，为了方便维护，除了 common 目录存放公用组件外，页面级别的组件也是按照和 views 中的子目录一一对应的方式组织的。

css 的组织方式，我们采用的是就近原则。即组件自己独有的 css 文件直接放在该组件的目录下，这样在后续的开发中，能很方便的定位到相关的 css 文件。另外在 assets 目录里有一个单独的 common.css 文件。这个文件包含的内容大多是全局的样式，比如整体布局样式、页面通用样式等。common.css文件从app.vue中全局引入。

登录以及权限控制

登录

用户在输入用户名和密码之后点击登陆，如果登录成功，服务端会返回一个该用户token，它也可以理解成是唯一标示用户身份的令牌。后端和我们约定好一个规则：用户登录之后的每一个 http 的请求都需要在请求头中带上这个 token，不带 token 或者 token 过期，接口都会返回401状态码。前端在用户登录之后会把 token 存储在本地的 cookie 里，以便后续的操作获取 token 的值。

权限控制

后台登录的每个角色都是有一定的权限的，普通用户就不能使用管理员才允许使用的功能。每个 http 请求，后端会根据请求头中的 token 来判定该用户的权限，如果用户没有权限，则返回403状态码。前端根据不同的状态码，来弹出不同的提示框。我们通过在响应拦截器中添加判断来达到这个功能：

其实，仅仅使用这种方式来进行权限控制并不是一个完整的方案。可以想像，登录之后，所有页面对所有用户均是可见的，无论有没有某一个页面的权限，都能看到这个页面。只不过当点击进入这个没有权限的页面时，用户无法做任何操作，置会看到一些没有数据的布局，然后页面上方会有弹窗提示“您没有此项操作的权限”，用户体验不是很好。理想的权限控制状态，是对页面进行权限控制，有某种角色的用户只能看到特定的页面。

要达到理想状态的页面控制，我们做了一些调研，尝试了一些方法，有一个具体的思路，就是在前端自己的路由控制文件中，把每个路由标识出可访问的权限。当用户登录之后，通过 token 获取用户的角色，动态根据角色分配不同路由，再通过 router.addRoutes 动态挂载路由。router.addRoutes 是 vue-router2.2.0 添加的方法，可以动态的添加新的路由到原有的路由表中。不过项目还没有进行代码层面的实现，在之后的优化安排中应该会落实下来。

结语

这篇分享主要是对SDMK-ADMIN项目从结构和工程的层面进行了一些梳理，另外在页面实战的层面也趟了蛮多的坑，比如不同页面组件之间的通信，状态管理，多图表页面的组织方式等等。但是受限于篇幅的原因，以后有机会的话再详细介绍。
