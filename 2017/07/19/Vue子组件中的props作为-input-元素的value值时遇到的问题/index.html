直接运行上面的代码，打开控制台会发现，vue会给出报错：

[Vue warn]: Avoid mutating a prop directly since the value will be overwritten whenever the parent component re-renders. Instead, use a data or computed property based on the prop’s value. Prop being mutated: “val”.

这是因为 props 是单向绑定的：当父组件的属性变化时，将传导给子组件，但是不会反过来。这是为了防止子组件无意修改了父组件的状态。

但是我们会发现 <input> 的内容会保持不变，也就是永远为空。这是因为 data 属性初始化的时候赋值了 this.val。可是在初始化的时候 val 的值是空。所以一秒之后即便是改变了 val 的值，inputValue 也不会更新了，因为根本没有地方去更新它。所以在上面的业务需求下，这种方法不适用。但是其他在非异步的情况下是可以使用这种方法的。
